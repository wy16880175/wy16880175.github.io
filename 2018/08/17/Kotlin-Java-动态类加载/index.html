Kotlin源码热部署加载

2018-08-17 17:00:19

前言

本着拿着锤子看什么都是钉子的精神，一直以来我热衷于把Kotlin推到各种场景中去，最近Web方面做的比较多，所以就来看看Kotlin能在Web上做到什么程度。一直以来，Web应用中比较常用的技术有php，java，.net(已死)，很明显，java能上的地方Kotlin都是能上的。但是如果不开发企业级应用，追求更快的更新，更便捷的开发的话，java的SSM明显不是最好的选择。这也是为什么php，python，nodejs这样的脚本越来越火的原因，免编译、热部署这样的特性真是很让人羡慕呢。
那么是否有办法让Kotlin实现类似的功能呢？

Kotlin Script

Kotlin摊子铺的很大。

Kotlin最初的目标就是实现一门针对工程的全栈语言。于是就有了Kotlin-jvm、Kotlin-js、Kotlin-native以及Kotlin Script。可能在看到Script之后，很多人会想到Kotlin是不是通过Javascript的分支实现脚本运行的呢。其实不是，Kotlin-js做的挺烂的。编译速度极慢，跟同为Js超集的Typescript完全不能比，与Js的交互也做的不如Ts那么平滑。

Kotlin的kts本质还是运行在JVM上的，一定要说有什么核心技术的话，也只是利用了JVM的动态类加载机制。每当运行kotlinc运行kts的时候，其实就是一个将kts文件编译成class然后启动JVM动态加载class的过程。

那么其实kts已经完全可以在Web后台实现类似php的功能了，我们只需要给每个路由规定一个kts,再由Nginx这样的服务端程序运行这些脚本就能实现热更新了。但是，这里有一个问题，每一个kts的运行都是需要开启一个新的JVM的，而启动一个JVM的代价是非常沉重的，尤其是对于我们的路由来说。往往一个路由消耗的资源还远远不到启动一个JVM占用的资源。所以直接使用kts是行不通的，我们需要自己实现一个kotlin源文件的加载器。

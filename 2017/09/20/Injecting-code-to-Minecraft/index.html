Injecting code to Minecraft

1.利用Instrumentation & javassist 监听Minecraft方法

开发Minecraft mod也有一段时间了，因为妄想开发一个通过p2p网络实现联机的mod，所以遇到了很多其他moder不太会遇到的问题。打算写一个系列的文章，记录一下这段时间遇到的各种问题和各种解决方案。

#引言
因为要实现自己的联机方式，所以就要对Minecraft中的各种变化做一个监听，其中最重要的也就是监听Block的变化。Minecraft forge提供了几个BlockEvent来满足一般modder对Block监听的需求，但是通过调试会发现这些BlockEvent的触发者其实不是Block，而是Player，也就是说Forge所提供的BlockEvent并不能监听爆炸，命令方块，红石等游戏元素造成的Block变化。这种时候就只能依赖更加低级的监听了，我选择去监听net.minecraft.world.World.setBlockState(…)方法。

#Javassist字节码操作库
javassist是个非常好用的字节码操作库。javassist相对于ASM的优势在于程序员不必了解java的字节码是怎么样的，虽然很多java程序员对字节码都有不错的了解。下面写一段摘自javassist tutorial的代码来展示一下javassist的便利

javassist可以通过这种非常类似于反射的操作来完成对指定类，指定方法的字节码操作。并且可以看到

插入的代码与真正的java代码没有任何的区别。这个很像php中的eval函数，并且也是在运行时编译的。

其实如果注入代码的目标不是Minecraft client的话，仅仅依靠javassist就能完成对方法的注入。

javassist的CtClass类提供了一个writeFile的方法，这个方法的原理是通过ClassLoader寻找所需修改的类的class文件，并直接修改该文件。但是Minecraft client的启动方式比较复杂，通过writeFile无法找到类对应的Class文件也就没法完成修改。这个时候就需要instrument的帮助了

#Instrumentation 代理
java中代理是个很常见的词，但是Instrumentation就没有那么常见了。Instrumentation是可以独立于应用程序之外的代理，它可以检测所有类的加载，并操作这些类。
这样的一个代理(Agent)由两个部分组成：
1.premain

2.transform

关于Premain-class的详细规范我就不多说了。这里主要看premain和transform具体做了什么。premain中的代码很简单，一行打印和一行添加transformer。在为Instrumentation添加了Transformer之后，每次加载新的Class时，就会先运行所add的transformer，也就是JInjection。再来看JInjection，这个类实现了ClassFileTransformer这个接口，如果你想要修改新加载的类，你就只需要在transform方法中返回修改后的字节码，如果不修改则返回null。
